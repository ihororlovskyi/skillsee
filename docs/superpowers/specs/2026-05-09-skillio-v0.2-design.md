# skillio v0.2 — design

## Summary

v0.2 reorients the CLI around three views of the skill universe (lock + on-disk in `.claude/skills` and `.agents/skills`):

- `skl` (= `skl cost`, `skl co`) — sorted-by-cost ballast view with a colored verdict on total ambient context spent on skill descriptions.
- `skl ls` — compact one-liner per source for quick name scanning, with a diff line surfacing stale lock entries.
- `skl usage` (= `skl us`) — usage × cost for an integrated consumption view, including "missed" rows for installed-but-unused skills. The previous `audit` command is renamed to `usage` (no compatibility alias).

`skl rm` becomes destructive on the filesystem (deletes skill directories), gated by a `Y/n` prompt and `--yes`.

This is a breaking release: the bare `skl` no longer runs audit, `skl rm` now touches disk, and `skl ls` output format changes.

## Non-goals

- Tab-completion for `skl rm` (deferred to TODO.md).
- Auto-prune via `npx skills rm` (deferred to TODO.md).
- A "writing-skills observer" experiment that watches sessions and proposes skill updates — out of scope for skillio itself; tracked separately as a process experiment.
- A new tokenizer dependency. Token estimation stays on `chars / CHARS_PER_TOKEN` (zero-dep operating rule).

## Architecture

A single discovery utility unifies the three sources, and each command is a thin renderer over the same data.

### `src/utils/discover-skills.ts` (new)

```ts
export interface SkillRecord {
  name: string;
  sources: Array<'lock' | '.claude' | '.agents'>;
  skillFile?: string;            // first existing path in priority order
  frontmatterTokens?: number;    // undefined = no SKILL.md or no frontmatter
  status: 'ok' | 'missing' | 'no-frontmatter';
}

export interface DiscoverInput {
  isGlobal: boolean;
  cwd: string;
  lockPath: string;
}

export function discoverSkills(input: DiscoverInput): Map<string, SkillRecord>;
```

Behavior:

- Reads lock at `lockPath`.
- For local scope: scans `<repoRoot>/.claude/skills/*/SKILL.md`, and additionally `<repoRoot>/.agents/skills/*/SKILL.md` if that directory exists.
- For global scope (`-g`): scans `~/.claude/skills/*/SKILL.md` and `~/.agents/skills/*/SKILL.md`.
- Merges by skill name. `sources` lists every place the skill appears.
- `skillFile` resolves via existing `findSkillFile` priority order.
- `status`:
  - `ok` — has frontmatter and tokens computed.
  - `missing` — name in lock but no SKILL.md anywhere.
  - `no-frontmatter` — SKILL.md exists but has no `---` block.

This replaces ad-hoc per-command scans. Today only `cost` does any discovery; `list` reads only the lock.

### `src/utils/ansi.ts` (new)

Tiny color helpers around ANSI 32/33/31. Disabled when `!process.stdout.isTTY` or `process.env.NO_COLOR` is set or `--no-color` is passed.

```ts
export function green(s: string): string;
export function yellow(s: string): string;
export function red(s: string): string;
export function setColorEnabled(enabled: boolean): void;
```

### `src/utils/confirm.ts` (new)

Synchronous Y/n via `node:readline/promises`, returns boolean. Defaults to N on empty input. No prompt when stdin is not a TTY (returns false — caller should require `--yes` for non-interactive).

### Command files

| Command | File | Status |

### CLI dispatch (`src/cli.ts`)

```
SUBCOMMAND_NAMES = { list, ls, remove, rm, cost, co, usage, us }
bare skl → costCommand.run(args)
```

The current `if (process.argv[2] === 'audit') argv.splice(...)` block is deleted. `skl audit` will be reported by citty as an unknown command.

## UX specs (per command)

### `skl` / `skl cost` / `skl co`

```
claude-md-improver       ~103 tok
brainstorming             ~58 tok
caveman                   ~52 tok
vitest                    ~47 tok
ghost-skill              missing
no-fm-skill              (no frontmatter)

Total: ~779 tok across 15 skills    OK — keep it lean
```

- **Universe:** all skills from any of lock / `.claude/skills` / `.agents/skills`.
- **Sort:** numeric tokens descending; `missing` and `no-frontmatter` rows at the bottom (alphabetical among themselves).
- **Total math:** sum of numeric tokens only.
- **Verdict text + color (ANSI 32/33/31, plus the verdict line itself):**
  - `< 1000` → green, `OK — keep it lean`
  - `1000 .. 1500` → yellow, `time to plan some cleanup`
  - `> 1500` → red, `ballast — clean it up`
- **Flags:** `-g`, `--json`, `--no-color`.
- **JSON:** `{ skills: [{name, tokens|status}], total, verdict: 'ok'|'plan'|'cleanup' }` — no ANSI.

### `skl ls` / `skl list`

```
.claude/skills    : brainstorming caveman vitest                              3 skills, ~157 tok
.agents/skills    : (empty)
skills-lock.json  : brainstorming caveman claude-md-improver executing-plans  4 skills, ~218 tok

skills-lock.json has 2 skills missing on disk: claude-md-improver, executing-plans
```

- **Per-source row:** label, then space-padded names, then `<count> skills, ~<tokens> tok`.
- **Tokens for lock-row:** sum of `frontmatterTokens` for those lock skills that exist on disk somewhere. Lock-only entries contribute 0 to tokens but 1 to count.
- **Empty source:** show `(empty)` when the source exists but has zero skills. When the source directory does not exist at all (typical for `.agents/skills` in a non-global repo), omit the row entirely — there is nothing to be transparent about.
- **Diff lines (each emitted only if non-empty):**
  - `skills-lock.json has N skills missing on disk: <names>` — lock-only entries.
  - `.claude/skills has N skills not in lock: <names>` — disk-only in claude.
  - `.agents/skills has N skills not in lock: <names>` — disk-only in agents.
- **No colors** (no `--no-color` flag — it has no effect).
- **Flags:** `-g`, `--json`.
- **JSON:** `{ '.claude/skills': [...], '.agents/skills': [...], 'skills-lock.json': [...], diffs: { lockOnly, claudeNotInLock, agentsNotInLock } }`.

### `skl usage` / `skl us`

```
Scope: /Users/me/work/myrepo  (use -g for global)
claude-code skill usage all-time (attributed)
Files read: 47; JSONL lines read: 12480

91 subagent-driven-development        ~ 58 tok * 91     = ~ 5278 tok
34 brainstorming                      ~ 47 tok * 34     = ~ 1598 tok
12 vitest                             ~ 52 tok * 12     = ~  624 tok
 0 caveman                            ~ 41 tok missed   = ~    0 tok
 0 npm-package                        ~ 38 tok missed   = ~    0 tok
 0 ghost-skill                        missing
 0 no-fm-skill                        (no frontmatter)

Total: ~7500 tok across 7 skills (consumption)
```

- **Sort:** by `count` descending; among ties → skill name ascending. `missing` / `no-frontmatter` rows go to the bottom (alphabetical).
- **Missed row format:** literal `missed` instead of `* <count>`, with `= ~ 0 tok` tail. Note the spaces around `~` and `*` to match the user's specified formatting.
- **Universe:** union of audit-derived skill names AND all skills from `discoverSkills` (so installed-but-unused skills appear as `missed`).
- **Multi-agent:** when `-a` is omitted, both agents render as separate blocks (header + rows + per-agent Total). No combined Total.
- **Flags:** `-p`, `-a`, `-g`, `--since`, `--mode`, `--scan-all-files`, `--root`, `--format json`.
- **No colors** (no `--no-color` flag — it has no effect).
- **JSON:** existing audit JSON shape, with each row gaining `tokensPerSkill` (number | null) and `consumption` (number).

### `skl rm` / `skl remove`

```
$ skl rm verification-before-completion
Will remove "verification-before-completion":
  - skills-lock.json
  - .claude/skills/verification-before-completion/  (3 files, ~52 tok)
  - .agents/skills/verification-before-completion/  (not found)

Proceed? [y/N] y

Removed "verification-before-completion" from skills-lock.json
Removed "verification-before-completion" from .claude/skills (3 files)
Skipped .agents/skills (not found)
```

- **Multi-skill:** `skl rm a b c` lists the plan for all three, then a single `Proceed? [y/N]`.
- **Y/n:** default N. Skipped under `--yes/-y` and under `--dry-run`.
- **`--dry-run`:** prints the plan, no prompt, no writes.
- **Edge cases:**
  - Skill present nowhere → `"<name>" is not in lock or on disk` then exit 1.
  - In lock only → still proceed; report `Skipped .claude/skills (not found)` etc.
  - On disk only → still proceed; report `Skipped skills-lock.json (not in lock)`.
  - SKILL.md directory contains extra files (scripts, resources) → recursive delete.
- **Path safety:** abort with non-zero exit if a target path resolves outside `homedir()` and outside the detected repo root. This guards against a corrupted lock file or symlink shenanigans pointing to `/etc` or similar.
- **Order of operations:** lock first (atomic write), then `.claude/skills/<name>/`, then `.agents/skills/<name>/`. A failure mid-run leaves the rest unchanged and returns non-zero. No rollback (the lock change is the cheap one to redo).
- **Output messages:** match the user-specified pattern — `Removed "<name>" from skills-lock.json`, `Removed "<name>" from .claude/skills`, etc. (Two short lines per skill, plus skip lines.)
- **Drop the trailing `JSON.stringify(Object.keys(...))` from the current remove.ts.** Pure noise.

## Color and TTY behavior (cross-cutting)

- ANSI enabled only when stdout is a TTY AND `NO_COLOR` env is unset AND `--no-color` not passed.
- `--json` always disables color regardless of TTY.

## Migration notes (CHANGELOG-style)

- `audit` renamed to `usage`. The old keyword (which was silently stripped before) is gone — run `skl usage` (or `skl us`).
- `skl` (no args) used to run audit; now runs `skl cost`. For the old usage view, run `skl usage`.
- `skl rm` now deletes files from `.claude/skills/<name>/` and `.agents/skills/<name>/`. Use `--dry-run` to preview, `--yes` for non-interactive.
- `skl list` output format changed. JSON shape changed. Use `--json` for stable output.
- New aliases: `skl us` (usage), `skl co` (cost).
